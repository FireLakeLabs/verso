2. What was lost

Content present in the plan but not represented in the PRD.

Dropped features (no PRD user story)

  • #13 Reader Affinity Network — Phase 4 force-directed cluster graph with bridge-book analysis. The plan's lowest-scoring item (V−E = −3). May be an intentional omission, but worth confirming.
  • #24 Narrator Playlist Builder — Phase 2 narrator-driven queueing, runtime running-total, audible:// deep links. The plan tied this to long-road-trip listening; PRD has no equivalent.

Dropped or diminished detail

  • Effort/value scoring (1–10 with derived Score = V−E) — entire scoring table and the score column are gone. PRD has no prioritization signal beyond Solid v1 anchoring.
  • Phased roadmap with checkboxes — Phases 1–4 with per-feature E/V annotations are reduced to a narrative paragraph under "Further Notes".
  • "High-leverage items worth highlighting" — the rationale for why #30, #22, #8, #27, #29 punch above their weight is not in the PRD.
  • Per-feature tech-stack choices — Recharts, Nivo, D3, react-calendar-heatmap, fuse.js, dnd-kit, TipTap/Lexical, react-pdf, html-to-image, Framer Motion, node-vibrant, @vercel/og, Hangfire, MJML, TimescaleDB, Meilisearch, ntfy.sh, Tesseract.js, Polly, CsvHelper, YamlDotNet, react-wordcloud, Cytoscape.js, React Force Graph. PRD collapses to "fit-for-purpose visualization libraries" + named TanStack Table.
  • Per-feature external API specifics — Open-Meteo weather overlay, Google Books fallback genres, Wikidata SPARQL author enrichment, ListenNotes narrator podcast cross-ref, Hardcover GraphQL specifically, NYT Books API list history, Open Library author API, Goodreads CSV-only post-API-closure note, Adobe PDF Extract API option, OpenAI embeddings cost note. PRD names a centralized auth layer but no per-feature pairing.
  • Audible field reference table — the response_groups field inventory at the top of the plan is gone. PRD speaks of "full source metadata" abstractly.
  • AudibleApi attribution — plan links rmcrackan/AudibleApi; PRD names "AudibleApi" without attribution.
  • Specific award list for #11 (Hugo, Nebula, Booker, Pulitzer, Edgar) and the sci-fi-bias framing are gone.
  • Wrapped personality archetype rule and shareable image card are gone (#15).
  • Lorenz / Pareto / Gini coefficient vocabulary for #3 abstracted to "author diversity score". The statistical framing is lost.
  • Mermaid timeline / Recharts ScatterChart visualization options for #12 are gone.
  • Wallpaper export and "polished personal tool" framing for #26 are gone.
  • Three-column board (drop / finish / defer) interaction model for #17 is gone — PRD only requires completion-band grouping.
  • HashiCorp Vault + Ubuntu homelab deployment context from stack notes is gone — PRD says "centralized external API auth layer" without naming Vault or the homelab.
  • Natural deployable cuts table — five named cuts (Weekend v1 / Solid v1 / Ambient companion / Integrated platform / Public release) — survive in compressed form under "Further Notes" but lose the per-cut rationale.
  • Open questions list — plan's five open questions (hosting, auth scope, mobile capture, snapshot cadence, public profile location) survive verbatim in "Further Notes" but framed as resolved-later rather than as the planning hand-off they were.

3. What is new in the PRD (not in the plan)

New material

  • Library ingestion as a first-class user-facing capability (US 1–2). In the plan, ingestion lives only in "Stack notes" as a precondition. The PRD promotes it to its own user stories and to an Implementation Decision that ingestion preserves source metadata broadly rather than normalizing.
  • Architectural guardrails as user stories (US 59–60) — "external API integrations isolated from core library views" and "AI features delayed until the core data model is stable". The plan implies these by phasing but never states them.
  • Testing Decisions section — an entire new dimension: contract-style fixtures for the AudibleApi boundary, heavy export testing, rules-engine testing for Smart Shelves, controllable clocks for background jobs, graceful failure for external adapters, structured-output fixtures for AI, privacy-controls tests before any sharing feature ships. None of this is in the plan.
  • Explicit Out of Scope list — payments/SaaS packaging, replacing Audible playback, manual editing of source metadata, native mobile apps, realtime sync, multi-user, AI-for-Solid-v1, public profile for Solid v1. The plan never enumerates non-goals.
  • Source-of-truth separation as a stated decision — "Keep source Audible facts separate from user-authored data... Audible metadata can be refreshed; tags, shelves, collections, journal entries, ratings, acknowledgements, and privacy preferences belong to Verso." The plan implies this by mentioning per-feature DB storage; the PRD elevates it to a principle.

4. Summary

  • Feature coverage is near-complete: 31 of 33 plan features map to PRD user stories. Only #13 Reader Affinity Network and #24 Narrator Playlist Builder are unrepresented.
  • The PRD trades implementation specificity for architectural discipline. Most loss is in named libraries, named external APIs, and per-feature tech choices. Most addition is in testing strategy, scope boundaries, and source-of-truth separation.
  • Prioritization signal weakened: the plan's E/V/Score model and phase checkboxes are gone. Solid v1 survives as the anchor target, but per-feature priority within and beyond Solid v1 is no longer encoded.
  • Recommended follow-ups: (a) decide whether #13 and #24 are intentional cuts or PRD omissions; (b) decide whether the plan's scoring table should remain a living artifact alongside the PRD; (c) preserve per-feature external-API and viz-library notes somewhere the PRD can reference, since they will resurface as implementation questions.
